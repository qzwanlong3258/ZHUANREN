1、[[预先危险性分析PHA]]
（1）操作

| 分析时间 | 在设计、施工、生产等活动之前，生命周期初始阶段 |
| ---- | ----------------------- |
| 分析人员 | 系统所涉及的各专业的工程师或专家        |
| 分析方法 | 头脑风暴                    |
(2) 分析流程
[[预先危险性分析PHA流程为什么要这样设计]]  

| 第1步 | 熟悉系统，确定系统将要保护的对象，通常为人、机、环三个方面。                                    |
| --- | ----------------------------------------------------------------- |
| 第2步 | 建立PHA计划，确定风险可接受水平。                                                |
| 第3步 | 确定PHA小组成员，由所涉及的各专业专家、工程师以及操作人员组成。                                 |
| 第4步 | 收集资料，了解相似系统或相关系统的情况。                                              |
| 第5步 | 辨识系统中存在的危险，分析每一个危险将要危及的对象。                                        |
| 第6步 | 评估每一个危险对每一个目标影响的严重程度以及发生概率。                                       |
| 第7步 | 根据风险评估结果决定风险可否接受，如果风险不可接受，是否提出风险控制措施，风险控制措施的优先顺序也应确定。             |
| 第8步 | 提出风险控制措施后，要对系统重新进行评估已确定采用控制措施过程是否出现新的危险，如新风险不可接受，则需重新制定控制措施，重新评估。 |
| 第9步 | 汇总分析结果并形成文件。                                                      |
2、[[故障假设分析]]WIA
① 首先要熟悉系统，确定要分析的范围，然后提出要分析的问题。
② 分析过程采用头脑风暴，分析中应确定假设出现某故障时其可能导致的
最坏的后果，列出所有的后果。
③ 检查和分析系统在设计初始阶段针对该故障已经采取的安全措施，判断
其是否能够真正阻止危险或风险降低到可接受的水平。
④ 若上一步不足以保证生产安全，则需要进一步的控制措施。
